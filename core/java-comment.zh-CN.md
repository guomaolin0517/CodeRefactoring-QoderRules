# 命名约定规范 v1.0
# ============================================
# AI 辅助代码注释的命名标准
# 通过将 [ENABLED] 更改为 [DISABLED] 来启用/禁用约定，规则或者约束默认ENABLED，除非特别标注DISABLED
#
# 使用方法：
# 1. 将此文件放在项目根目录
# 2. 根据您的语言/框架启用/禁用约定
# 3. 在 AI 对话中使用 @njava-comment.zh-CN.txt 引用
# 4. AI 将只遵循 ENABLED 的约定
# 5.本规范适用于所有 Java 后端开发项目，无关业务领域与行业属性，是保障代码一致性、可维护性、安全性的基础准则
# 最后更新：2026-03-11
# ============================================

> **【强制】** 类注释：包含类功能、作者、创建日期。

```java
/**
 * 用户管理Controller，负责用户CRUD操作
 * @author 张三
 * @date 2024-05-01
 */
```

> **【强制】** 方法注释：包含功能描述、参数说明、返回值说明、异常说明。

```java
/**
 * 查询用户信息
 * @param userId 用户ID
 * @return UserDTO 用户信息
 * @throws BusinessException 无此用户
 */
```

> **【强制】** 复杂逻辑注释：关键业务逻辑、条件判断、循环体需添加注释，说明设计思路。

> **【禁止】** 无意义注释（如 `// 循环`、`// 赋值`），**禁止**注释过时代码（直接删除）。
