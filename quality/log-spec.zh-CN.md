### [规则 1] 日志选型与配置

> **【强制】** 使用 **SLF4J + Logback** 框架，通过 `LoggerFactory.getLogger(XXX.class)` 或 `@Slf4j` 注解注入，**禁止**直接使用 Log4j/JDK Logging API 或 `System.out.println`。

> **【强制】** 日志配置文件命名为 `logback-spring.xml`，统一输出格式：

```
%d{yyyy-MM-dd HH:mm:ss.SSS} - [%X{traceID}] - [%thread] %-5level %logger{50}.%M(%line) - %msg%n
```

> **【强制】** 日志文件命名：`应用名.log.yyyy-MM-dd`，文件大小限制 **50-100MB**，按日期滚动切割，保留时间根据业务需求配置（建议不少于 30 天）。

### [规则2 ] 日志级别与使用场景

| 级别 | 使用场景 | 说明 |
|------|----------|------|
| **FATAL** | 系统不可用（如数据库连接失败） | 需立即人工介入，极少使用 |
| **ERROR** | 功能异常（如业务处理失败、外部服务调用失败） | 记录完整上下文（参数、异常堆栈） |
| **WARN** | 潜在风险（如参数格式不规范、接口即将废弃） | 无需人工介入但需关注 |
| **INFO** | 核心业务节点（如用户登录、订单提交、服务启动完成） | 不冗余（占比不超过日志总量 10%） |
| **DEBUG** | 开发调试（如方法入参出参、循环细节） | 生产环境禁用 |

> **【禁止】** 日志中打印敏感信息（身份证、手机号、密码、银行卡号），需脱敏：
- 手机号：`138****5678`
- 身份证：`110101****1234`

### [规则3 ] 日志编写要求

> **【强制】** 日志内容包含 "操作主体 - 操作行为 - 结果 - 关键参数"。

```java
logger.info("用户{}提交订单，订单号{}，状态{}", userId, orderNo, status);
```

> **【强制】** 字符串拼接使用占位符，**禁止**字符串拼接（避免无日志级别判断时的性能损耗）。

```java
// 正确
logger.debug("用户{}查询数据，参数{}", userId, param);

// 错误
logger.debug("用户" + userId + "查询数据，参数" + param);
```

> **【强制】** 分布式链路日志需关联追踪 ID（`traceID`），便于跨服务问题排查。
